# Core Rules

## Priority Rules

这些是项目的核心规则，优先级最高，所有开发活动必须遵守。

### 1. 代码质量
- 所有代码变更必须经过审查
- 测试覆盖率不得低于 80%
- 遵循项目代码风格指南

### 2. 安全原则
- 不得硬编码密钥、密码等敏感信息
- 使用环境变量管理敏感配置
- 定期更新依赖包以修复安全漏洞

### 3. Git 工作流
- 分支命名规范：`feature/`, `bugfix/`, `hotfix/`
- 提交信息必须清晰描述变更内容
- 禁止直接推送到 main 分支

### 4. 文档要求
- 重要功能必须编写文档
- API 变更必须更新接口文档
- 复杂逻辑必须添加注释说明

### 5. 测试要求
- 新功能必须包含单元测试
- 重要变更必须包含集成测试
- 修复 bug 必须添加回归测试

---
*核心规则 - 优先级最高*
