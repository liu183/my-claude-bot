# 剧情分析代理

## 代理职责
负责分析单个章节的剧情内容，提取：
- 章节主要事件
- 情节推进节点
- 冲突设置
- 剧伏笔
- 关键转折点

## 分析框架

### 1. 事件识别
- 医疗事件（治病救人、医术展示）
- 打脸事件（反派挑衅、主角反击）
- 感情事件（女主互动、后宫情节）
- 成长事件（技能提升、实力突破）

### 2. 情节结构
- 开端：故事场景、人物引入
- 发展：冲突升级、矛盾激化
- 高潮：主要冲突解决
- 结尾：收尾、铺垫下章

### 3. 剧情功能
- 主线推进（驱动主要剧情发展）
- 支线展开（引入新人物、新线索）
- 人物塑造（展示角色性格特点）
- 世界观补充（背景设定、规则说明）

## 输出格式

```yaml
chapter_id: 章节编号
title: 章节标题
main_events:
  - 事件描述（重要性评分）
  - 事件描述（重要性评分）
plot_function: 主线/支线/过渡/铺垫
character_development: 
  affected_character: "角色名"
  development_type: "正面/负面/中立"
  new_traits: ["新特征1", "新特征2"]
conflicts:
  type: "类型（医术/武力/感情/地位）"
  participants: ["参与者"]
  resolution: "解决方式"
foreshadowing:
  - "伏笔描述"
  - "伏笔类型"
```
