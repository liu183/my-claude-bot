## 平台规则

### 1. 签约流程和要求

**签约条件：**
- 完成实名认证
- 作品字数达到2万/5万/8万字时各有1次签约申请机会
- 3次签约评估全部失败的作品将不支持签约
- 申请签约期间作品无更新要求

**签约流程：**
1. 获得签约资格（字数达标+实名认证）
2. 安全审核（7天内，工作日进行）
3. 签约评估（7天内，评估题材、节奏、人设、开篇、主线）
4. 填写合同（合同签约人必须是账号实名认证人）
5. 联系责编开放签约
6. 电子签约（手机验证码）

**签约评估维度：**
- 题材是否适合签约
- 节奏、人物设定、开篇、代入感、主线
- 是否存在不安全、低俗、言语不通、故事性不强、侵权等底线问题

### 2. 稿费和福利机制

**稿费构成公式：**
```
稿费 = 阅读收益 + 听书收益 + 互动收益 + 集团渠道收益 + 集团版权收益 +
       全勤收益 + 买断/保底 + 渠道收益 + 版权收益 + 其他收益
```

**听读收益：**
- 主要来自广告收益分成
- 不同广告类型和结算方式收益不同
- 阅读人数和阅读深度相同时收益也会波动

**互动收益：**
- 读者打赏产生
- 结算时间：每个自然月的第二个周四24点前
- 分成比例：去除运营商渠道费后的七三分成（作者70%）
- 不同系统打赏渠道费不同（iOS和安卓差异）

**全勤奖规则（2025年更新）：**
1. 签约作品更新且有效过审字数满10万字的次月起才可获得
2. 自然月内听书收益与阅读分成收益之和≥500元
3. 每日有效更新字数满4000字或6000字
4. 未通过审核的章节不计入打卡字数
5. 保底作品需上月超保底才能参与

**完本续签奖：**
- 常规完结作品：分基础档、进阶档、优质档
- 奖金类型：完本续签奖励（不可叠加）+ 新书连更奖（可叠加）
- 巅峰榜作品：额外激励，无金额上限

### 3. 推荐规则

**多书名实验：**
- 签约后可设置至多5款书名+封面组合
- 实验时间：7-10天产出最优书名
- 必须与内容人设、题材保持一致，不能骗点击
- 书名要有区分度，换风格、换卖点

**推荐启动：**
- 签约作品达到8万字时可主动开启推荐
- 稳定更新影响推荐效果
- 断更会导致读者流失，影响数据

### 4. 平台功能使用

**定时发布：**
- 设置路径：电脑端【工作台】-【作品管理】-【发文编辑器】-【定时发布】
- 距离发布不足30分钟时系统锁定无法修改
- 章节按顺序排队，插队需先取消前面定时
- 审核一般24小时内完成，建议提前存稿

**章节调序：**
- 发布超过3天的章节不能调序
- 第一卷前三章不能调序
- 番外章节不能调序
- 24小时内只有一次调序权限
- 需番茄作家助手APP升级到330及以上版本

**多书名实验：**
- 设置路径：【作品运营】-【推荐素材管理】-【多书名实验】
- 书名字符不超过15字，支持中英文、数字、部分标点
- 需一次性上传所有实验组（最多5个），实验中不可增加
- 实验结果产出后72小时内选择是否使用最优书名

### 5. 审核规则

**审核不通过原因：**
- 内容不适合收录
- 低俗色情
- 学生负面导向
- 格式未分段
- 信息不符

**审核要求：**
- 双休日及国家法定节假日不进行审核
- 工作日审核一般7天内完成

## 写作技巧

### 1. 开篇技巧

**黄金三章法则：**
- 第一章：抛出核心冲突
- 第二章：展现金手指
- 第三章：明确短期目标

**开篇要素：**
- **人物**：通过动作与场景展示主角人设
- **背景**：时间、地点、冲突层面，延续简介/文名核心梗
- **场景**：连续时空中的冲突动作，让人物生活发生转折变化

**开篇禁忌：**
- 开篇拖沓、平淡或信息轰炸
- 大量堆砌背景介绍、人物琐事
- 塞入大量世界观设定、人物背景
- 切入点缺乏吸引力，采用常规老套方式

**开篇技巧：**
- 直接从冲突开始写
- 场景不能复杂（1-3个场景足够）
- 人物不能太多（开篇3个人物足够）
- **悬疑式开篇**：制造信息差
- **人设式开篇**：短文字内让读者喜欢上主角
- **思路式开篇**：巧妙构思，新颖脑洞
- **复仇式/目标式开篇**：血海深仇、困境+目标

### 2. 钩子设置

**钩子定义：**
- 所有伏笔、期待感、情绪值，一切能吸引读者继续看下去的内容

**钩子类型：**
- **悬疑式钩子**：信息差制造悬念
- **情感类钩子**：约定、承诺（需剧情人设做足）
- **复仇式钩子**：三年之期已到
- **养成式钩子**：系统、期待值+1

**钩子设置技巧：**
- **开头钩**：大刀阔斧，无需铺垫就吸引读者
- **章节钩**：章末留下悬念，装了一半的逼、突然跳出来的反派

**九连环钩子：**
- 钩子要环环相扣
- 每章末尾设置小钩子
- 平淡章节末尾加猛料
- 主角回家坐着一个死去的人

### 3. 剧情设计

**核心设计思路：**

**时间锁+空间锁：**
- 时间锁：倒计时、限时任务
- 空间锁：限制空间内制造紧张感（风雪山庄）
- 两者结合使用效果1+1>2

**道德挑战：**
- 给主角设置道德困境
- 生死、利益、友情、个人vs集体的抉择
- 表现人物内心，突显人设

**大人物遇小事，小人物遇大事：**
- 大人物遇小事：信息差、身份反差感
- 小人物遇大事：迪化文，普通人的逆袭

**情感共鸣：**
- 从七宗罪找灵感（傲慢、嫉妒、暴怒、懒惰、贪婪、暴食、色欲）
- 转化为积极美德：傲慢→自信，嫉妒→好胜，暴怒→勇气
- 用情节和描写烘托情绪，不用情绪形容词

**期待感形成：**
- 未满足的情绪落差
- 危机制造：生理需求、安全需求、社交需求、尊重需求
- 金手指：反差和获得，在危机时引入
- 关系的改变：从误解到信任，从仇视到原谅

### 4. 人设塑造

**人设设计要点：**
- "不妥协"：给角色设计绝不妥协的点
- 坏人要坏到脚底流脓，好人要好到让人恶心
- 把单一人物特点写到极致

**人设避雷：**
- 人设扁平化：所有角色说话语气相同
- 前后不统一：开头杀伐果断，后面优柔寡断
- 人设与情节矛盾：行为不符合既定人设
- 配角工具人化：只有功能没有性格

**人设塑造方法：**
- 通过事件发展去描写人物行动、思考、反应
- 建立人物小传：身份、年龄、职业、性格、欲望、目标、恐惧
- 保持人设在故事情节里的前后统一性
- 人设的成长与变化要在逻辑合理范畴内

### 5. 大纲设计

**大纲重要性：**
- 理清思路，防止崩剧情
- 防止人物性格OOC
- 给自己留下想象空间

**大纲写法：**

**四分法：起承转合**

**五分法推荐：**
1. 破题、开头、发展、高潮、结尾
2. 目标、障碍、行动、意外、结果

**破题五分法详解：**
- **破题**：开篇用剧情破题，让读者明白要讲什么故事
- **开头**：快速吸引读者注意，带冲突矛盾的剧情
- **发展**：主角利用手段往上爬
- **高潮**：一系列陷害、落难、示弱、表白后的爆发
- **结尾**：主角达成目标，翘起来的结尾

**目标五分法示例：**
- **目标**：女主带领家人逃离抄家危机
- **障碍**：皇族忌惮，联手陷害
- **行动**：利用金手指保全财产
- **意外**：男主被打断双腿
- **结局**：脱离困境，身份平反，发家致富

**人物关系表：**
- 必须做，短则几十万，长则几百万字记不住
- 记录名字、与主角关系、性格、年龄
- 后期用到不用回头翻章节

### 6. 节奏控制

**节奏管理要点：**
- 稳定更新：每天写、持续写、不中断
- 结构节奏：在哪一章抛出冲突，在哪一段落推进节奏
- 前期有钩子，中段能起伏，后期能升华
- 写得出爆点，更要守得住节奏

**避免节奏混乱：**
- 成长太快、报应太慢
- 单剧情太漫长
- 换地图太频繁
- 冲突乏力、爽点不足

**节奏控制技巧：**
- 增加冲突矛盾：设置阻碍与挑战
- 合理布局：关键节点设置冲突和悬念
- 增强爽感：激烈矛盾+递进铺垫+冲突细节
- 注意情节过渡自然流畅

## 避雷点

### 1. 新手常见错误

**写作人称混乱：**
- 一会儿第一人称，一会儿第三人称
- 新手建议用第三人称，第一人称视角局限，写不长

**流水账问题：**
- 晚上下班→吃饭→看直播→有人敲门
- 解决办法：增加冲突（前因后果）、加入强情绪

**开篇不吸引人：**
- 开篇太寡淡：准备不充分，没想好超级吸引人的噱头
- 开篇太复杂：场景太多、人物太多，读者混乱

**人设有问题：**
- 人设扁平化：角色说话语气相同
- 前后不统一：开头大女主，后面大圣母

**故意秀文笔：**
- 词藻堆砌，华丽描写过多
- 故事要简洁，用最简单的形容词
- 一到两个精准形容词就足够

### 2. 审核不通过原因

**内容问题：**
- 内容不适合收录
- 低俗色情
- 学生负面导向
- 格式未分段

**信息问题：**
- 合同信息与认证信息不一致
- 填错手机号、银行卡号

### 3. 读者负面反馈原因

**轻毒点：**
- 成长太快
- 报应太慢
- 人设偏离
- 单剧情太漫长
- 换地图太频繁

**重毒点：**
- 太弱智
- 太圣母
- 总被虐
- 反三观
- 拆CP

**五过度：**
- 过度调皮变小丑
- 过度深情变油腻
- 过度高冷变傲慢
- 过度深刻变说教
- 过度善良变圣母

### 4. 多书名实验常见问题

**流量没变化：**
- 书名没有区分度
- 换汤不换药，风格太像

**流量反而下降：**
- 书名骗点击
- 噱头太足，内容不对胃口
- 吸引来的人留不住

**书名审核不通过：**
- 使用了英文标点符号
- 书名含字符超过15字

### 5. 封面简介常见问题

**封面问题：**
- 底图、书名不清晰
- 要素不突出（缺书名、作者名）
- 排版过度拥挤
- 字号分级不合理

**简介问题：**
- 文不对题：题材和简介不匹配
- 文艺病：堆砌辞藻无重点
- 流水账：剧透全文无悬念
- 自嗨式：忽略读者代入感

## 可设计的 Agent Skills

基于以上提取的知识点，以下是可以抽象为 Agent Skills 的具体技能：

### 1. 题材评估与选择技能
**功能描述：** 根据市场热点和作者兴趣，评估题材可行性和市场潜力
**适用场景：** 开书前选题阶段
**输入：** 作者兴趣、市场趋势数据
**输出：** 题材建议、市场分析、竞争度评估

### 2. 开篇优化技能
**功能描述：** 检查开篇是否符合黄金三章法则，提供优化建议
**适用场景：** 开篇写作、签约前优化
**输入：** 开篇前三章内容
**输出：** 冲突设置评估、金手指展现评估、目标明确性评估、优化建议

### 3. 钩子设置检测技能
**功能描述：** 检测章节末尾是否有有效钩子，提供钩子设置建议
**适用场景：** 每章写作完成后
**输入：** 章节内容
**输出：** 钩子识别、钩子类型、钩子强度评估、改进建议

### 4. 人设一致性检查技能
**功能描述：** 检查人物行为是否符合既定人设，发现人设矛盾
**适用场景：** 长篇写作中、修改阶段
**输入：** 人设设定、章节内容
**输出：** 人设一致性报告、矛盾点定位、修改建议

### 5. 节奏控制分析技能
**功能描述：** 分析故事节奏，发现节奏问题（拖沓、过快）
**适用场景：** 中段剧情、整体结构审查
**输入：** 多章节内容
**输出：** 节奏曲线、冲突密度分析、爽点分布、调整建议

### 6. 简介生成技能
**功能描述：** 根据故事内容生成符合黄金结构的简介
**适用场景：** 开书前包装、多书名实验
**输入：** 故事核心梗、人设、主线
**输出：** 多版本简介（开场定调版、高能开场版、钩子收尾版）

### 7. 封面设计建议技能
**功能描述：** 根据书名、题材提供封面设计建议
**适用场景：** 开书前包装、推荐素材优化
**输入：** 书名、题材、目标读者
**输出：** 配色方案、字体建议、元素搭配、排版建议

### 8. 大纲生成技能
**功能描述：** 根据故事核心生成结构化大纲（五分法）
**适用场景：** 开书前准备
**输入：** 故事核心梗、主角设定
**输出：** 破题、开头、发展、高潮、结尾五部分大纲

### 9. 流水账检测技能
**功能描述：** 检测是否写成流水账，提供改进方案
**适用场景：** 写作过程中
**输入：** 章节内容
**输出：** 流水账识别、冲突分析、前因后果检查、改进方案

### 10. 多书名生成技能
**功能描述：** 为同一作品生成多个有区分度的书名
**适用场景：** 多书名实验、书名测试
**输入：** 故事内容、核心卖点、目标读者
**输出：** 3-5个不同风格的书名及理由

### 11. 人设生成技能
**功能描述：** 根据角色定位生成完整人设
**适用场景：** 人物设定阶段
**输入：** 角色定位、故事需求
**输出：** 人物小传（身份、年龄、职业、性格、欲望、目标、恐惧、不妥协点）

### 12. 情节冲突设计技能
**功能描述：** 根据需要设计包含时间锁、空间锁、道德挑战的情节
**适用场景：** 卡文、平淡章节、高潮设计
**输入：** 当前剧情、人物设定
**输出：** 冲突设计方案（时间锁/空间锁/道德挑战）

### 13. 毒点检测技能
**功能描述：** 检测文中是否存在常见毒点
**适用场景：** 写作完成后、签约前
**输入：** 章节内容
**输出：** 毒点类型、严重程度、位置定位、修改建议

### 14. 世界观构建技能
**功能描述：** 协助构建完整的世界观体系
**适用场景：** 架空背景、奇幻题材
**输入：** 故事类型、时代背景
**输出：** 地理、历史、文化、政治、科技、社会体系设定

### 15. 预告关键词生成技能
**功能描述：** 为定时发布章节生成读者预告关键词
**适用场景：** 定时发布设置
**输入：** 章节内容、下章预告
**输出：** 吸引人的预告关键词（3-5个）

### 16. 签约评估模拟技能
**功能描述：** 模拟签约评估，预测签约可能性
**适用场景：** 申请签约前自测
**输入：** 前两万字内容
**输出：** 签约可能性评估、问题诊断、改进方向
