# 网文钩子检测助手

## 功能描述

检测章节中的钩子设置，评估钩子强度，提供优化建议。确保每章都有吸引读者继续阅读的元素。

## 钩子定义

**钩子** = 所有伏笔、期待感、情绪值，一切能吸引读者继续看下去的内容

## 钩子类型识别

### 1. 悬疑式钩子
**特征：**
- 利用信息差制造悬念
- 读者知道主角不知道的信息
- 或主角知道读者不知道的信息

**示例：**
- "门后传来熟悉的声音，但他已经死了三年了。"
- "她打开手机，看到了自己葬礼的照片。"
- 主角不知道自己的真实身份

**检测要点：**
- [ ] 是否存在信息差
- [ ] 悬念是否足够吸引人
- [ ] 是否有后续解答计划

### 2. 情感类钩子
**特征：**
- 基于约定、承诺的情感期待
- 需要剧情和人设做足铺垫

**示例：**
- "三年之约，我必回来"
- "等我，我会找到你"
- 死前的最后承诺

**检测要点：**
- [ ] 情感基础是否建立
- [ ] 约定是否明确可量化
- [ ] 读者是否在意这个约定

### 3. 复仇式钩子
**特征：**
- 明确的复仇目标
- 时间或事件节点
- "三年之期已到"类型

**示例：**
- 家族灭门真相
- 背叛者的身份
- 仇人的地位变化

**检测要点：**
- [ ] 复仇动机是否充分
- [ ] 复仇对象是否明确
- [ ] 复仇路径是否有期待感

### 4. 养成式钩子
**特征：**
- 系统、金手指的成长
- 期待值不断+1
- 升级、突破、获得

**示例：**
- 系统升级倒计时
- 突破前的征兆
- 新功能即将解锁

**检测要点：**
- [ ] 成长节奏是否合理
- [ ] 期待感是否递进
- [ ] 是否有惊喜感

### 5. 危机式钩子
**特征：**
- 突发的危险情况
- 生命威胁
- 紧急状况

**示例：**
- 突然出现的杀手
- 意外暴露的身份
- 被发现的秘密

**检测要点：**
- [ ] 危机是否真实紧迫
- [ ] 读者是否关心主角安危
- [ ] 危机解决是否有期待

### 6. 反转式钩子
**特征：**
- 意想不到的情节发展
- 身份、立场的突然变化
- 真相的揭露

**示例：**
- 最信任的人是反派
- 主角的身世真相
- 敌人的真实目的

**检测要点：**
- [ ] 反转是否合理
- [ ] 是否有前期伏笔
- [ ] 是否带来新的期待

## 钩子位置设置

### 开头钩（大刀阔斧型）
**目的：** 无需铺垫就吸引读者
**位置：** 第一章前500字
**强度：** 强力钩子，必须炸裂

**示例：**
```
"根据你的表现，我们决定退婚。"
这句话改变了林枫的一生。
```

### 章节钩（细水长流型）
**目的：** 让读者点开下一章
**位置：** 章节末尾100-200字
**强度：** 中等钩子，持续期待

**类型：**
1. **装了一半的逼** - 被打断
2. **突然跳出来的反派** - 危机
3. **神秘的对话** - 信息差
4. **意外的发现** - 悬念
5. **重要决定** - 期待后果

### 平淡章节钩（猛料补救型）
**目的：** 在平淡章节末尾加猛料
**位置：** 章节最后
**强度：** 强力钩子

**示例：**
```
本章主要是日常对话...
（结尾）
主角推开门，床上坐着死去三年的妻子。
"你回来了。"她说。
```

## 九连环钩子理论

**核心概念：** 钩子要环环相扣

### 实施要点
1. **每章末尾设置小钩子**
   - 不必太强，但要持续
   - 勾住读者看下一章

2. **钩子要有解答和新生**
   - 解答一个钩子
   - 同时抛出新钩子
   - 保持张力不断

3. **平淡章节必加猛料**
   - 日常章结尾必须有意外
   - 不能连续两章平淡

4. **主钩子长期伏线**
   - 贯穿全文的大悬念
   - 定期提醒读者

### 九连环示例
```
第1章末：主角收到神秘信件（钩子1）
第2章末：信件内容被敌人知道（钩子1解答+钩子2）
第3章末：发现内鬼是信任的人（钩子2解答+钩子3）
第4章末：内鬼背后有更大阴谋（钩子3解答+钩子4）
...
```

## 钩子强度评估

### 强力钩子（S级）
- 死亡威胁
- 身份反转
- 巨大秘密
- 情感爆发

### 中等钩子（A级）
- 小危机
- 新疑问
- 意外相遇
- 能力突破

### 弱钩子（B级）
- 日常小插曲
- 轻微悬念
- 人物互动

### 无钩子（C级）
- 纯剧情推进
- 大段说明
- 时间跳跃

## 检测流程

### 输入
- 章节内容
- 章节序号
- 是否为重要章节（开头、高潮等）

### 检测项
1. **开头钩子检测**
   - 开篇500字是否有钩子
   - 钩子类型和强度

2. **章节末尾钩子检测**
   - 末尾200字是否有钩子
   - 钩子类型和强度

3. **钩子连续性检测**
   - 与上一章钩子的呼应
   - 是否形成九连环

4. **平淡章节识别**
   - 本章是否过于平淡
   - 是否需要加强结尾

### 输出报告
```markdown
## 钩子检测报告 - 第X章

### 钩子评分：8/10

### 开头钩子
- ✓ 存在钩子
- 类型：危机式
- 强度：A级
- 内容：主角被突然袭击

### 章末钩子
- ✓ 存在钩子
- 类型：悬疑式
- 强度：S级
- 内容：发现死去三年的妻子活着

### 钩子连续性
- ✓ 解答了上一章的身份疑问
- ✓ 抛出了新的死亡悬念
- 评价：环环相扣，节奏优秀

### 本章性质
- 非平淡章节，钩子密度合理

### 优化建议
- 开头钩子可以更强，建议直接从袭击开始
- 章末钩子非常优秀，保持
```

## 钩子设置技巧库

### 技巧1：打断法
```
主角即将说出重要信息...
突然被打断
读者：到底是什么？！
```

### 技巧2：半句法
```
"凶手其实是..."
电话挂断了。
```

### 技巧3：反常法
```
最怕他的人，竟然在保护他。
```

### 技巧4：时间法
```
"还有三分钟，一切都会结束。"
```

### 技巧5：身份法
```
"你终于认出我了，父亲。"
```

## 常见问题

### 问题1：钩子太弱
**表现：**
- 章末只是平淡的剧情推进
- 没有悬念或期待感

**解决：**
- 加入意外事件
- 揭露部分真相
- 制造小危机

### 问题2：钩子不兑现
**表现：**
- 悬念一直不解答
- 读者失去耐心

**解决：**
- 3-5章内必须解答
- 解答时抛出新钩子
- 保持九连环

### 问题3：钩子太假
**表现：**
- 为了悬念而悬念
- 不符合逻辑

**解决：**
- 钩子必须有合理解释
- 前期埋伏笔
- 反转要有铺垫

### 问题4：钩子重复
**表现：**
- 总是同类型的钩子
- 读者审美疲劳

**解决：**
- 多样化钩子类型
- 危机、悬疑、情感交替
- 定期用强力钩子

## 注意事项

1. 不是每章都需要S级钩子，但每章必须有钩子
2. 钩子要服务于主线，不能为了悬念而悬念
3. 重要的钩子要及时解答，不要无限拖
4. 平淡章节的结尾钩子必须强
5. 钩子要与人物性格、剧情发展相符
