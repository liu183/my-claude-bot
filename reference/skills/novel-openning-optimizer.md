# 网文开篇优化助手

## 功能描述

检查网文开篇是否符合黄金三章法则，识别常见问题并提供优化建议。

## 黄金三章法则

### 第一章：抛出核心冲突
- ✓ 必须有明确的冲突事件
- ✓ 主角面临困境或转折
- ✓ 读者能感知到故事的张力
- ✗ 避免大段背景介绍
- ✗ 避免日常琐事堆砌

### 第二章：展现金手指
- ✓ 展示主角的特殊能力/优势
- ✓ 给予读者期待感
- ✓ 与第一章冲突产生联系
- ✗ 避免金手指设置太弱
- ✗ 避免金手指与主线无关

### 第三章：明确短期目标
- ✓ 主角有明确的行动方向
- ✓ 读者知道主角要做什么
- ✓ 埋下后续剧情钩子
- ✗ 避免目标模糊
- ✗ 避免目标与主线脱节

## 开篇要素检查

### 人物检查
- [ ] 主角通过动作与场景展示人设
- [ ] 开篇人物不超过3个主要角色
- [ ] 每个角色有明确出场目的
- [ ] 角色对话符合人设

### 背景检查
- [ ] 时间地点清晰明确
- [ ] 冲突层面交代清楚
- [ ] 与简介/文名核心梗呼应
- [ ] 没有过量世界观设定

### 场景检查
- [ ] 场景在1-3个之间
- [ ] 场景连续不跳跃
- [ ] 有明确的冲突动作
- [ ] 人物生活发生转折

## 常见问题诊断

### 问题1：开篇拖沓
**症状：**
- 前三章没有实质性冲突
- 大量环境描写或心理活动
- 故事进展缓慢

**解决方案：**
- 删减前500字，直接从冲突开始
- 合并前两章内容
- 在第一章末尾加入强悬念

### 问题2：信息轰炸
**症状：**
- 开篇塞入大量设定
- 人物关系复杂难记
- 世界观解释过多

**解决方案：**
- 将设定融入剧情逐步展现
- 减少开篇人物数量
- 删除独立的世界观说明段落

### 问题3：缺乏吸引力
**症状：**
- 切入点平淡无奇
- 采用常规老套路
- 读者无继续阅读欲望

**解决方案：**
- 使用四种开篇技巧之一：
  - 悬疑式：制造信息差
  - 人设式：短文字让读者喜欢主角
  - 思路式：新颖脑洞
  - 复仇/目标式：困境+目标

### 问题4：人称混乱
**症状：**
- 第一人称和第三人称混用
- 视角频繁切换

**解决方案：**
- 新手统一使用第三人称
- 一个场景内保持单一视角
- 视角转换要有明确分隔

## 开篇技巧库

### 技巧1：直接冲突法
```
炸了一栋楼开始写
死了一个人开始写
被追杀开始写
```

### 技巧2：悬念式开篇
```
主角醒来发现自己变成了...
主角发现家里坐着一个人
主角收到了自己死后的照片
```

### 技巧3：反差式开篇
```
最强的高手在送外卖
最大的反派在卖红薯
最穷的乞丐继承亿万资产
```

### 技巧4：对话式开篇
```
"你只有三天时间了。"
"这是最后的机会。"
"签了它，或者死。"
```

## 使用流程

1. **用户提供前三章内容**
2. **系统逐章检查**
   - 第一章：核心冲突检查
   - 第二章：金手指展现检查
   - 第三章：短期目标检查
3. **要素完整性评估**
   - 人物要素
   - 背景要素
   - 场景要素
4. **问题诊断与建议**
   - 识别存在的问题
   - 提供具体修改方案
   - 给出优秀案例参考

## 输出格式

### 评估报告示例

```markdown
## 开篇评估报告

### 整体评分：7.5/10

### 第一章评估
- ✓ 核心冲突明确（主角被退婚）
- ✗ 冲突强度偏弱
- 建议：增加家族压力，提高冲突紧迫感

### 第二章评估
- ✓ 金手指展现清晰（系统觉醒）
- ✗ 金手指与主线关联不够
- 建议：系统任务与退婚复仇直接关联

### 第三章评估
- ✓ 短期目标明确（参加试炼）
- ✗ 缺少后续钩子
- 建议：章末加入神秘人物或意外事件

### 要素完整性
- 人物：8/10（主角人设清晰，配角略单薄）
- 背景：7/10（时间地点明确，世界观设定偏多）
- 场景：9/10（场景集中，冲突动作明显）

### 优化建议
1. 第一章：删除前300字的家族介绍，直接从退婚场景开始
2. 第二章：系统首任务改为"三天内通过试炼"，与主线关联
3. 第三章：结尾加入神秘老者出现，埋下后续钩子
```

## 注意事项

1. 开篇不要追求完美，重要的是快速吸引读者
2. 不同题材对开篇要求不同，灵活应用
3. 修改开篇时注意与后续剧情的连贯性
4. 可以参考同类题材的开篇优秀案例
