# 第一篇章 Multi-Agent 创作系统 - 使用指南

> **系统版本**：v1.0
> **创建时间**：2025-03-15
> **适用范围**：第1-100章正文创作
> **系统目标**：模块化解耦、长篇上下文一致性、质量可控

---

## 📖 系统概述

本Multi-Agent创作系统为第一篇章（1-100章）设计，通过9个专业Agent协同工作，实现高质量、高效率、高一致性的网文创作。

### 核心优势

1. **模块化解耦**：每个Agent独立负责特定功能，便于维护和升级
2. **长篇一致性**：通过共享参考材料和验证机制，确保100章逻辑一致
3. **质量可控**：三层验证体系，确保输出质量稳定
4. **并行创作**：部分Agent可并行工作，提高创作效率
5. **可扩展性**：Agent可独立升级，不影响其他模块

---

## 🤖 Agent架构

### 创作层Agent（6个）

```
Agent 1: 情节架构师 (Plot Architect)
    ↓
Agent 2: 对话大师 (Dialogue Master)
Agent 3: 场景描绘师 (Scene Painter)
Agent 4: 动作导演 (Action Director)
Agent 5: 人物塑造师 (Character Sculptor)
Agent 6: 情感编织者 (Emotion Weaver)
    ↓
整合内容 → 形成初稿
```

### 验证层Agent（3个）

```
初稿
    ↓
Agent 7: 一致性检查员 (Consistency Checker) ← P0级，一票否决
    ↓
Agent 8: 质量审查员 (Quality Reviewer)
Agent 9: 风格校准员 (Style Calibrator)
    ↓
最终定稿
```

## 🔄 创作工作流程

### 完整流程图

```
Step 1: 准备阶段
├── 读取章节细纲
├── 读取时间线、人物档案、场景设定
└── 确认前后章节摘要

Step 2: 架构阶段 (Agent 1)
├── 构建情节框架
├── 设计关键场景（3-5个）
├── 标注转折点
└── 输出情节骨架

Step 3: 创作阶段 (Agent 2-6，可并行)
├── Agent 2: 创作对话
├── Agent 3: 描写场景
├── Agent 4: 设计动作
├── Agent 5: 塑造人物
└── Agent 6: 编织情感

Step 4: 整合阶段
├── 汇总所有Agent输出
├── 按情节框架整合
└── 形成初稿（2000-2500字）

Step 5: 验证阶段 (Agent 7-9，串行)
├── Agent 7: 一致性检查（P0级）
│   ├── 时间线一致性
│   ├── 人物状态一致性
│   ├── 地理设定一致性
│   ├── 战力等级一致性
│   └── 物品连续性
│
├── Agent 8: 质量审查
│   ├── 文笔质量（30分）
│   ├── 情节逻辑（30分）
│   ├── 角色塑造（25分）
│   └── 节奏控制（15分）
│
└── Agent 9: 风格校准
    ├── 爽文节奏（25%）
    ├── 后宫文特色（25%）
    ├── 主角设定（20%）
    ├── 情感尺度（15%）
    └── 文风统一（15%）

Step 6: 优化阶段
├── 根据验证反馈修改
├── 二次验证
└── 最终定稿

Step 7: 输出阶段
├── 生成正文（2000-2500字）
├── 生成元数据（人物状态更新）
├── 生成摘要（前后衔接用）
└── Git提交
```

## 🎯 质量保证机制

### 三级验证体系

#### 一级验证：Agent 7（一致性检查员）
- **标准**：100%通过，P0级问题零容忍
- **检查项**：时间线、人物状态、地理设定、战力等级、物品连续性
- **一票否决**：发现P0级问题，立即停止，必须修改

#### 二级验证：Agent 8（质量审查员）
- **标准**：8.0分以上（10分制）
- **评分维度**：文笔（30分）、情节（30分）、角色（25分）、节奏（15分）
- **处理方式**：低于7.0分需要重写，7.0-7.9分建议修改

#### 三级验证：Agent 9（风格校准员）
- **标准**：90%以上符合度
- **评分维度**：爽文节奏、后宫文特色、主角设定、情感尺度、文风统一
- **处理方式**：低于70%需要大改，70-79%需要调整

### 人工审核节点

- **每完成10章**：阶段审核，检查整体质量
- **每完成30章**：阶段总结，调整创作方向
- **完成100章**：整篇审核，准备发布

## 🔧 使用指南

### 开始创作新章节

1. **准备阶段**
   ```bash
   # 确认章节号
   CHAPTER_NUM=5

   # 读取参考材料
   - 005_细纲/第1-30章_详细细纲.md（查阅第5章细纲）
   - 003_时间线/第一篇章_完整时间线.md（确认Day 5）
   - 002_人物档案/第一篇章_人物档案.md（确认苏婉儿登场）
   - 004_场景设定/第一篇章_主要场景设定.md（确认医院场景）
   ```

2. **执行Agent 1（情节架构师）**
   - 调用Agent 1配置
   - 输入：细纲、时间线、人物档案
   - 输出：情节框架

3. **并行执行Agent 2-6**
   - 调用Agent 2-6配置
   - 输入：情节框架、各自参考材料
   - 输出：对话、场景、动作、人物、情感

4. **整合内容**
   - 汇总所有Agent输出
   - 按情节框架整合
   - 形成初稿

5. **执行Agent 7-9验证**
   - 依次调用Agent 7、8、9
   - 根据反馈修改
   - 最终定稿

6. **输出章节**
   - 保存正文到：`007_正文/第5章_温柔护士苏婉儿.md`
   - 更新元数据
   - Git提交

### Agent调用示例

#### 调用Agent 1（情节架构师）
```
请根据agent配置/plot_architect_config.md的指令，创作第5章的情节框架。

输入材料：
- 章节：第5章《温柔护士苏婉儿》
- 时间：末世后第5天
- 地点：临海市医院
- 核心事件：收服苏婉儿
- 登场人物：林天、苏婉儿

请输出情节框架。
```

#### 调用Agent 7（一致性检查员）
```
请根据agent配置/consistency_checker_config.md的指令，检查第5章的一致性。

输入材料：
- 章节正文：[粘贴正文]
- 时间线：003_时间线/第一篇章_完整时间线.md
- 人物档案：002_人物档案/第一篇章_人物档案.md

请输出一致性检查报告。
```

## ⚠️ 常见问题

### Q1: 如果Agent 7发现P0级问题怎么办？

**A**: 立即停止创作流程，修复问题后重新验证。P0级问题包括：
- 时间线错误
- 人物状态严重错误
- 战力严重崩坏
- 地理设定严重错误

### Q2: 如果质量评分低于8.0分怎么办？

**A**: 根据Agent 8的反馈修改，重点改进低分项，修改后重新审查。

### Q3: 如果风格符合度低于70%怎么办？

**A**: 根据Agent 9的反馈调整，重点改进偏差项，调整后重新校准。

### Q4: 如何提高创作效率？

**A**:
1. 熟练掌握Agent配置
2. 准备好所有参考材料
3. 使用快捷命令和脚本
4. 批量创作相似章节

### Q5: 如何确保100章一致性？

**A**:
1. 严格使用时间线和人物档案
2. 每章都进行一致性检查
3. 定期（每10章）回顾检查
4. 保持详细的创作日志
