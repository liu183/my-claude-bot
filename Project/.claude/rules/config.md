# Configuration Rules

## Config Management Rules

### 1. 配置文件管理
- 所有配置文件必须版本控制
- 敏感配置使用环境变量
- 生产配置与开发配置分离

### 2. 配置变更流程
1. 在配置文件中添加变更说明
2. 测试环境验证
3. 制定回滚方案
4. 生产环境变更

### 3. 配置迁移
- 使用 `config-migration` skill 进行迁移
- 确保迁移过程可回滚
- 记录迁移历史

### 4. 配置验证
- 配置变更后必须验证
- 使用 schema 验证配置格式
- 检查配置项依赖关系
